# DELEGATE FEATURE WORKFLOW

> This workflow defines how the Orchestrator (Gemini CLI), following direction from the Manager (Claude), delegates the implementation of a new feature to specialized workers.

## 1. Planning (Manager / Orchestrator)
- Claude defines high-level requirements and architecture.
- Gemini CLI analyzes specific impact on Frontend and Backend.
- Identify the impact on both Frontend and Backend.
- Define specific tasks for `@frontend-worker` and `@backend-worker`.
- Create a detailed plan in `PLAN.md` or a dedicated feature doc.

## 2. Delegation (Execution)
- **Backend Implementation:**
  - Invoke `@backend-worker` with the task description, relevant schemas, and service layer patterns.
  - Target: `backend/app/services/`, `backend/app/api/`, `backend/app/models/`.
- **Frontend Implementation:**
  - Invoke `@frontend-worker` with UI requirements, design system constraints, and API specs.
  - Target: `frontend/src/components/`, `frontend/src/services/`, `frontend/src/pages/`.

## 3. Review & Integration (Orchestrator)
- Inspect the code generated by workers.
- Run tests (if applicable) or verify structural integrity.
- Manually handle complex integration points that cross worker boundaries.
- Ensure compliance with `GEMINI.md` standards.

## 4. Finalization
- Update `HISTORY.md` with the completed feature details.
- Trigger `sync-docs.md` to update project documentation.
